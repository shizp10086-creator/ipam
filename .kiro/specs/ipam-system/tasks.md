# 实施计划：IT 智维平台（优化版）

## 概述

基于用户优化建议，采用**并行开发轨道 + MVP 优先 + 低代码引擎提前启动 + 事务编排骨架前置 + 测试左移**策略重构任务列表。

### 核心策略
- **接口契约先行**：脚手架完成后立即定义 API 契约和消息队列 Schema，前后端可基于 Mock 并行开发
- **MVP 优先**：第一波交付核心 IPAM 闭环（脚手架→认证→网段+IP→前端基础），用户能管理 IP
- **低代码引擎提前**：表单/报表引擎与业务模块并行开发，后续模块可通过配置生成页面
- **Saga 骨架前置**：事务编排框架尽早搭建，IPAM 分配 IP 联动 DHCP/DNS 需要
- **测试左移**：属性测试与业务逻辑同步编写，数据采集用 Mock Server 测试
- **复用开源**：Docker Compose 现成模板、Alembic + Factory_Boy、ipaddress/netaddr、Vue Element Admin Pro

### 执行节奏参考
- 第一周：Docker 脚手架 + 前端 Pro 模板 + 数据库设计 + API 契约
- 第二周：并行开发 — 后端写 Auth 和 IPAM Core；前端用 Mock 开发 IP 列表页
- 第三周：集成联调，跑通 IP 分配全流程
- 后续迭代：DCIM、NAC、数据采集等重资产模块

技术栈：Python 3.11 + FastAPI + SQLAlchemy 2.0 + Alembic + Celery + Vue 3 + Element Plus Pro + MySQL 8.0 + TimescaleDB + Redis + Elasticsearch + RabbitMQ

## 说明

- 任务标记 `*` 的为可选属性测试任务，可跳过以加速 MVP
- 每个 Phase 内标注了可并行的轨道，团队可根据人力分配并行推进
- Phase 1 是 MVP 核心闭环，优先保证用户能管理 IP
- 低代码引擎（Phase 2）提前启动，后续模块可通过配置生成页面，减少约 50% CRUD 代码
- Saga 事务编排骨架在 Phase 1 前置搭建，Phase 4 填充 DHCP/DNS 补偿逻辑
- PPT 生成引擎（需求 181）和 IT 价值量化放到 Phase 6 最后
- 数据采集引擎使用 Mock Server 测试，不真实 Ping 外部网络
- 属性测试使用 Hypothesis 库，最低 100 次迭代
- 单元测试使用 pytest + Factory_Boy，聚焦边界条件和错误路径
- 复用成熟开源方案：Docker Compose 模板、Alembic、ipaddress/netaddr、Vue Element Admin Pro
